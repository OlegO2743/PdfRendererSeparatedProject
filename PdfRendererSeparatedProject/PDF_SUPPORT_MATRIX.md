# PDF Support Matrix

Актуальная матрица поддержки PDF для проекта `PdfViewer.WinForms`.

Этот файл полезнее старого `README.md`, потому что отражает текущее состояние кода, а не исходный учебный замысел проекта.

Важно: сейчас проект является **PDF viewer / renderer**, а не полноценным **PDF editor**.

## Легенда

- `Да` — поддержка есть и реально используется
- `Частично` — поддержка есть, но не для всех случаев или не на уровне Acrobat/Chrome
- `Нет` — пока не реализовано

## 1. Версии PDF и общая совместимость

| Раздел | Статус | Комментарий |
|---|---|---|
| Классические PDF 1.0-1.4 без экзотики | Да | Обычно открываются лучше всего: обычные indirect objects, Flate/DCT, типовые шрифты, обычные страницы |
| PDF 1.5+ без object streams/xref streams | Частично | Многие такие файлы открываются, если объекты всё ещё лежат в файле в "плоском" виде |
| PDF 1.5+ с object streams как основным механизмом | Нет | Полноценной поддержки object streams нет |
| PDF с xref streams как обязательной частью структуры | Нет | Полноценного spec-level разбора xref streams нет |
| Linearized PDF | Частично | Нередко открываются, потому что парсер сканирует файл целиком, а не полагается только на xref |
| Incremental update / revision chain | Нет | Полноценной поддержки цепочек обновлений нет |

## 9. Практический вывод

Сейчас движок лучше всего подходит для:

- обычных текстово-графических PDF
- учебных, технических и офисных документов
- значительной части старых "классических" PDF без экзотических фильтров и document-level features

Пока самые слабые зоны такие:

- сложные формульные шрифты
- полное совпадение glyph mapping с браузером/Acrobat
- advanced color / transparency
- object streams / xref streams
- интерактивные возможности PDF

## 10. Что делать в первую очередь

Ниже приоритет уже не "по красоте архитектуры", а по тому, что чаще всего будет встречаться в реальных PDF и что сильнее всего заметно пользователю.

### Приоритет A — делать первым

| Что реализовать | Почему это важно |
|---|---|
| Точное отображение сложных embedded formula fonts | Для технических, научных и учебных PDF это критично. Сейчас это самая заметная проблема: формулы, индексы, спецсимволы, скобки, математические шрифты |
| Полноценная поддержка современных object streams / xref streams | Очень частый случай у PDF 1.5+ и новых генераторов PDF. Без этого часть современных файлов вообще не откроется |
| Поддержка page `/Rotate`, `/CropBox` и близких page boxes | Это регулярно встречается в реальных документах, особенно в сканах, презентациях и экспортированных PDF |

### Приоритет B — делать сразу после этого

| Что реализовать | Почему это важно |
|---|---|
| Расширить набор image filters: прежде всего `JPXDecode`, затем `LZWDecode`, дальше `JBIG2Decode` | Это часто влияет на сканы, журналы, старые PDF и документы с большим количеством изображений |
| Улучшить Type0/CID и CFF/Type1C совместимость до уровня "максимально близко к браузеру" | Это нужно не только для формул, но и для старых книг, статей, технических архивов |
| Transparency / ExtGState / blend support | Часто встречается в PDF из Adobe, Illustrator, InDesign, браузеров и современных офисных экспортов |

### Приоритет C — можно позже

| Что реализовать | Почему это можно отложить |
|---|---|
| Аннотации, ссылки, bookmarks | Полезно, но для базового просмотра страницы не всегда критично |
| Формы AcroForm | Важно не для всех документов |
| Encryption / signatures / tagged PDF / layers | Нужны, но обычно это уже следующий уровень viewer-а |

## 11. Приоритет именно для нашего набора файлов

Если ориентироваться не на абстрактный "средний PDF в мире", а на те PDF, которые мы уже реально открывали в проекте, то фактический порядок такой:

1. Добить формульные и специализированные embedded-шрифты
2. Добить точный glyph mapping и метрики текста
3. Добить page-level совместимость (`/Rotate`, boxes)
4. Добить современные object streams / xref streams
5. Добить дополнительные image filters

То есть да: по нашим текущим файлам проблема с формулами и формульными шрифтами — это действительно задача номер один.

## 12. На что смотреть дальше

Если двигаться уже по рабочему roadmap, то разумный порядок такой:

1. Добить точное отображение сложных embedded formula fonts
2. Добить точный glyph mapping и метрики формул/спецсимволов
3. Добавить page `/Rotate`, `/CropBox` и related boxes
4. Добить object streams / xref streams
5. Расширить набор image filters
6. Добавить transparency / blend support
7. Решать document-level features: annotations, links, forms, bookmarks
